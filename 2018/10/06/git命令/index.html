git配置

git config –global user.name “lys”
git config –global user.email “liu_yangshuo@163.com“

git获取版本库

git clone http://xxxxxx

git初始化一个新仓库

新建一个文件夹，然后在文件夹下执行：
git init
执行完命令就会在该文件夹下生成.git文件夹

git代码提交流程

新建或修改文件
使用git add命令将文件添加到缓存区
使用 git commit将修改文件提交到本地代码库
使用 git push将本地文件同步到远程代码库

git常用命令

git status //使用该命令可以判断git代码提交流程执行到哪一步，该命令显示当前的状态信息
git diff //该命令会显示当前已修改但是还没加入缓存区的内容
git diff –cached //该命令显示使用git add加入到缓存区中的文件内容
git commit -m “git log” //将加入到缓存区中的内容提交到本地版本库，
git commit -a im “git log” //如果不是用git add那么上面这个命令只会将修改的文件提交到版本库，不会提交新建文件
git branch //查看所有分支，以及当前所在分支
git branch branch_01_01 //新建一个分支
git checkout master //切换分支
git merge -m ‘merge branch_01_01 branch’ branch_01_01 //将branch_01_01分支合并到当前分支
git branch -d branch_01_01 //删除分支branch_01_01,该命令只能删除已经合并的分支，如果强制删除某个分支使用-D参数
git reset –head HEAD^ //撤销合并的内容
git log //查看所有日志
git log v1.1… main.cpp /source //该命令显示从v1.1开发source目录下所有main.cpp文件的修改
git log –stat //改命令显示日志更详细的内容
